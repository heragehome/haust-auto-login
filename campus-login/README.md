# 校园网自动认证部署指南

## 一、文件说明

| 文件 | 用途 |
|------|------|
| `NetLogin.ps1` | 主脚本，每次执行即检测+登录，执行完退出 |
| `config.json` | 账号配置，与脚本同目录 |
| `netlogin.log` | 运行日志，自动生成在脚本同目录 |

## 二、配置 config.json

```json
{
  "username": "2023123456",
  "password": "your_password",
  "isp": "wcmcc"
}
```

ISP 后缀对照：

| 值 | 运营商 |
|----|--------|
| `wcmcc` | 移动 |
| `unicom` | 联通 |
| `wtelecom` | 电信 |
| `whaust` | 校园网 |

## 三、任务计划程序部署

### 3.1 打开任务计划程序

按 `Win + R`，输入 `taskschd.msc`，回车。

### 3.2 创建任务

1. 右侧点击 **"创建任务"**（注意：不是"创建基本任务"）。

2. **"常规"选项卡**：
   - 名称：`校园网自动认证`
   - 选中 ✅ **不管用户是否登录都要运行**
   - 选中 ✅ **使用最高权限运行**
   - 配置为：`Windows 10`（或你的系统版本）

3. **"触发器"选项卡** → 点击 **"新建"**，分别创建两个触发器：（这两个触发器可以根据自己的需要选择，具体见文末**注意事项**）

   **触发器 1 — 网络状态变化事件：**

   - 开始任务：`当特定事件被记录时`
   - 日志：`Microsoft-Windows-NetworkProfile/Operational`
   - 源：`NetworkProfile`
   - 事件 ID：`10000`
   - 点击确定

   **触发器 2 — 定时重复：**

   - 开始任务：`按预定计划`
   - 选择 `每天`，时间设为 `00:00:00`
   - 勾选 ✅ `重复任务间隔`，选择 `5 分钟`
   - 持续时间：`无限期`
   - 点击确定

4. **"操作"选项卡** → 点击 **"新建"**：
   - 操作：`启动程序`
   - 程序或脚本：
     ```
     powershell.exe
     ```
   - 添加参数（可选参数）：
     ```
     -NoProfile -NonInteractive -ExecutionPolicy Bypass -WindowStyle Hidden -File "C:\Scripts\NetLogin.ps1"
     ```
     > ⚠️ 将路径替换为你实际存放 `NetLogin.ps1` 的位置。

5. **"条件"选项卡**：
   
   - ❌ 取消勾选 `只有在计算机使用交流电源时才启动此任务`（笔记本建议取消）
   - ❌ 取消勾选 `如果计算机改用电池电源则停止`
   
6. **"设置"选项卡**：
   
   - ✅ 勾选 `如果过了计划开始时间，立即启动任务`
   - ✅ 勾选 `如果请求后任务还在运行，强制将其停止`
   - `如果此任务已经运行` 选择 `不启动新实例`
   
7. 点击 **确定** 保存，输入你的 Windows 登录密码确认。

### 3.3 验证

1. 为防止刷屏，脚本在网络正常时运行无日志。需先打开飞行模式，右键刚创建的任务 → **"运行"**
2. 检查脚本目录是否生成了 `netlogin.log`
3. 查看日志内容确认是否正常 内容是  `[ERROR] - - 无法获取本机 IPv4 地址`

## 四、日志说明

### 4.1 日志格式



```tex
yyyy-MM-dd HH:mm:ss [状态] IP地址 运营商 详细信息
```



### 4.2 状态码说明

| 状态          | 含义         | 可能原因                                                     | 处理建议                                                     |
| :------------ | :----------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| `SUCCESS`     | 认证成功     | 正常                                                         | 无需操作                                                     |
| `ONLINE`      | 已在线       | 探测时已处于在线状态，或登录时返回“已经在线”                 | 无需操作                                                     |
| `FAIL`        | 认证失败     | 密码错误、运营商选择错误、账号欠费、账号未绑定当前 IP 等     | 检查 `config.json` 中的 `username` / `password` / `isp` 是否正确；确认账号运营商类型（`wcmcc` / `unicom` / `wtelecom` / `whaust`）；联系运营商确认账户状态 |
| `UNKNOWN`     | 未知响应     | 服务器返回了无法识别的 `result` 值                           | 查看日志中的 JSON 原文，排查是否为认证系统更新               |
| `PARSE_ERR`   | 响应解析失败 | 登录接口返回了非 JSONP 内容，或响应被篡改                    | 检查网络是否被劫持到其他页面，或认证服务器异常；可临时手动打开浏览器登录确认 |
| `NET_ERR`     | 登录请求异常 | 网络不通、认证服务器不可达、HTTPS 证书错误、超时             | 确认本机到 `wlan.haust.edu.cn:802` 的网络连通性（可尝试 `Test-NetConnection wlan.haust.edu.cn -Port 802`）；检查系统时间是否正确（影响 SSL）；若持续出现，检查防火墙/代理 |
| `WARN`        | 状态可疑     | 探测到 HTTP 200 但响应体极短（<100 字节）且不包含认证特征    | 记录该现象，一般不影响；若反复出现，可能为探测逻辑需更新（如新劫持页面特征） |
| `UNREACHABLE` | 网络不通     | 所有探测尝试（IPv4 和备用 IP）全部超时或连接失败             | 检查网线、Wi‑Fi 是否连接；检查 DHCP 是否正常获取 IP；确认网关/交换机无物理故障；重启网络适配器 |
| `PROBE_FAIL`  | 探测彻底失败 | 域名探测和备用 IPv4 固定地址均不可达，会额外记录主探测的错误消息（如连接重置） | 查看该条日志附带的错误详情；常见于本机无有效网络连接         |
| `ERROR`       | 脚本运行错误 | 无法获取本机 IPv4 地址                                       | 检查网络适配器是否有首选 IPv4 地址；禁用再启用网卡；重启系统 |
| `FATAL`       | 致命异常     | 配置文件读取失败、或其他未捕获的脚本异常                     | 查看日志中的具体错误信息；确认 `config.json` 存在且格式正确  |
| `MUTEX`       | 互斥锁跳过   | 同一脚本已有实例正在运行                                     | 正常保护机制，无需处理；若频繁出现，可适当延长计划任务间隔   |

### 4.3 常见问题排查

1. 日志显示 `FAIL`：`Rad:Mac, IP, NASip, PORT err(2)!`

- **原因**：运营商后缀（`isp`）与账号实际运营商不匹配。例如账号是移动，但 `config.json` 中填写了 `whaust`。
- **解决**：修改 `isp` 字段为对应的运营商后缀：移动 `wcmcc`，联通 `unicom`，电信 `wtelecom`，校园网 `whaust`。

2. 日志显示 `FAIL`，msg 为其他错误

- 可能是选错运营商、密码错误、账号被禁用、欠费等，建议登录自助服务页面检查账号状态。

3. 脚本运行后网络仍然无法访问互联网

- 确认日志中状态是否为  `NET_ERR` ： `登录请求异常: 操作超时。 ` 
  - 检查是否有防火墙/代理正在开启或异常关闭，关闭后重试

- 确认日志中状态是否为 `SUCCESS` 或 `ONLINE`。若为 `SUCCESS` 但依旧无法上网，可尝试：
  - 清空 DNS 缓存：`ipconfig /flushdns`
  - 重置浏览器代理
  - 使用 `curl` 或浏览器访问 `edge.microsoft.com/captiveportal/generate_204` 手动验证
- 若为 `UNREACHABLE`，先确认网卡已连接且获取到 IP（如 `10.9.x.x`）。

4. Windows 任务计划程序执行后无日志输出

- 日志在脚本运行时自动创建，为防止刷屏，脚本在网络正常时运行无日志。需先打开飞行模式，右键 `NetLogin.ps1` 选择 `使用PowerShell运行`
- 检查任务“操作”配置是否正确（程序为 `powershell.exe`，参数为 `-NoProfile -NonInteractive -ExecutionPolicy Bypass -WindowStyle Hidden -File "脚本实际路径"`，引号采用半角输入）。
- 检查脚本所在目录是否有写入权限。
- 尝试手动执行脚本观察输出。

5. 登录成功但极短时间内又被踢下线

- 校园网强制 24 小时踢下线一次，此为正常行为。脚本在下次触发时会自动重新登录。

## 五、如何卸载

- 如需删除，请直接删除 `campus-login` 整个文件夹。
- 在“任务计划程序”中选择 `任务计划程序库` 注意不是 `任务计划程序（本地）` 
- 主页中间 `名称` 一栏下滑找到你创建的任务，点击 `❌删除` 创建的任务

## 六、注意事项

- 自 Windows Vista 开始，微软就推出了触发器合并技术，将各个程序所需要的触发器 **合并** 或 **集中** 起来运行，**触发器 1 — 网络状态变化事件** 自身就兼任刷新任务栏网络状态等职责。
- 本脚本为轻量脚本，使用 **“事件”** 加 **“时间”** 双触发器驱动，借助Windows平台本身就需要维持的大量触发器运行，几乎不产生额外功耗
- 如果不追求长时间挂机，可以仅保留 **“事件”** 触发器，即**触发器 1 — 网络状态变化事件** 它仅会在每次开机、睡眠唤醒、网络断开或重连的时候运行一次
- 脚本不会驻留内存，每次由任务计划触发后执行完毕即退出
- **Edge 隐私政策原文：**
  
  > 如果出现网络问题，Microsoft Edge 会发出不带 Cookie 的请求：
  > http://edge.microsoft.com/captiveportal/generate_204
  > 检查重定向的响应代码。如果被重定向，Microsoft Edge 将在新选项卡页打开重定向目标。
- 脚本会完全模仿 Microsoft Edge 在 Windows 下发起强制门户检测时的 HTTP 请求，脚本发出的探测流与 Edge 自带的强制门户检测在应用层不可区分。
- 本项目的5min定时运行机制规律太过明显，未来可能有封禁风险，如介意请勿安装 **触发器 2 — 定时重复：** 
- 认证服务器处于内网，在开启防火墙/代理时不可达。如果认证失败请检查网络配置
- 本脚本适用于 **锐捷 ePortal V4/V5（Web Portal 认证）认证系统** 欢迎开发者进行迁移、二次开发
- 调试时可在主流程加 `Write-Output "IP: $localIP | Status: $status"` 查看探测结果
- 如果学校换了认证域名，需要修改脚本中的 `wlan.haust.edu.cn` 相关匹配
